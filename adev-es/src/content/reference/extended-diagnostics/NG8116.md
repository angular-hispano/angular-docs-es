# Directiva estructural faltante

Este diagnóstico garantiza que un componente standalone que usa directivas estructurales personalizadas (por ejemplo, `*select` o `*featureFlag`) en una plantilla tenga las importaciones necesarias para esas directivas.

```typescript
import { Component } from '@angular/core';

@Component({
  // La plantilla usa `*select`, pero no se importó la directiva correspondiente.
  imports: [],
  template: `<p *select="let data from source">{{ data }}</p>`,
})
class MyComponent {}
```

## ¿Qué está mal con eso?

Usar una directiva estructural sin importarla fallará en tiempo de ejecución, ya que Angular intentará vincular a una propiedad `select` del elemento HTML, que no existe.

## ¿Qué debo hacer en su lugar?

Asegúrate de que la directiva estructural correspondiente esté importada en el componente:

```typescript
import { Component } from '@angular/core';
import { SelectDirective } from 'my-directives';

@Component({
  // Agrega `SelectDirective` al array `imports` para que esté disponible en la plantilla.
  imports: [SelectDirective],
  template: `<p *select="let data from source">{{ data }}</p>`,
})
class MyComponent {}

```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`missingStructuralDirective` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingStructuralDirective": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
