# Sufijo no compatible

Este diagnóstico detecta cuando los sufijos `.px`, `.%` y `.em` se usan con un enlace de atributo.

```html
<img [attr.width.px]="5">
```

## ¿Qué está mal con eso?

Estos sufijos solo están disponibles para enlaces de estilo. No tienen ningún significado al vincular a un atributo.

## ¿Qué debo hacer en su lugar?

En lugar de usar los sufijos `.px`, `.%` o `.em` que solo son compatibles con enlaces de estilo,
mueve esto a la asignación de valor del enlace.

```html
<img [attr.width]="'5px'">
```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`suffixNotSupported` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "suffixNotSupported": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
