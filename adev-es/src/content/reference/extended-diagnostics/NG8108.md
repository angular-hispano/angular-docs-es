# `ngSkipHydration` debe ser un atributo estático

`ngSkipHydration` es un atributo especial que indica a Angular que un componente específico debe excluirse de la [hidratación](guide/hydration).
Este diagnóstico garantiza que el atributo `ngSkipHydration` se establezca de forma estática y que el valor sea `"true"` o un valor vacío.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `<user-viewer ngSkipHydration="hasUser" />`,
})
class MyComponent {
  hasUser = true;
}
```

## ¿Qué está mal con eso?

Como atributo especial implementado por Angular, `ngSkipHydration` debe ser analizablemente estático para que Angular sepa en tiempo de compilación si la hidratación es necesaria para un componente.

## ¿Qué debo hacer en su lugar?

Al usar `ngSkipHydration`, asegúrate de que se establezca como un atributo estático (es decir, no uses la sintaxis de enlace de plantilla de Angular).

```typescript
import { Component } from '@angular/core';

@Component({
  template: `
    <user-viewer ngSkipHydration />
    <user-viewer ngSkipHydration="true" />
  `,
})
class MyComponent {}
```

Si es necesario usar una condición, puedes envolver el componente en un `*ngIf`.

```html

import {Component} from '@angular/core';

@Component({
template: `

<div \*ngIf="hasUser; else noUser">
<user-viewer ngSkipHydration />
</div>

    <ng-template #noUser>
      <user-viewer />
    </ng-template>

`,
})
class MyComponent {}

```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`skipHydrationNotStatic` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "skipHydrationNotStatic": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
