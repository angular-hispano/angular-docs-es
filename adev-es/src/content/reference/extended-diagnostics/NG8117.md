# Las funciones deben invocarse en interpolaciones de texto.

Este diagnóstico detecta funciones no invocadas en interpolaciones de texto.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `{{ getValue }}`,
})
class MyComponent {
  getValue() {
    return 'value';
  }
}
```

## ¿Qué está mal con eso?

Las funciones en interpolaciones de texto deben invocarse para devolver un valor.
Si la función no se invoca, no devolverá ningún valor y la interpolación no funcionará como se espera.

## ¿Qué debo hacer en su lugar?

Asegúrate de invocar la función cuando la uses en una interpolación de texto para devolver el valor.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `{{ getValue() }}`,
})
class MyComponent {
  getValue() {
    return 'value';
  }
}
```

## Requisitos de configuración

[`strictTemplates`](/tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`uninvokedFunctionInTextInterpolation` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInTextInterpolation": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](/extended-diagnostics#configuración) para más información.
