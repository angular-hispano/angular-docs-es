# Cadena opcional no anulable

Este diagnóstico detecta cuando el lado izquierdo de una operación de cadena opcional (`.?`) no incluye `null` ni `undefined` en su tipo en plantillas de Angular.

```typescript
import { Component } from '@angular/core';

@Component({
  // Muestra el nombre del usuario de forma segura, incluso si `user` es `null` o `undefined`.
  template: `<div>User name: {{ user?.name }}</div>`,
})
class MyComponent {
  // `user` se declara como un objeto que _no puede_ ser `null` ni `undefined`.
  user: { name: string } = { name: 'Angelino' };
}
```

## ¿Qué está mal con eso?

Usar el operador de cadena opcional con una entrada no anulable no tiene efecto e indica una discrepancia entre el tipo permitido de un valor y cómo se presenta en la plantilla.
Un desarrollador podría asumir razonablemente que la salida del operador de cadena opcional podría ser `null` o `undefined`, pero nunca será ninguno de esos valores en realidad.
Esto puede generar confusión sobre la salida esperada del programa.

## ¿Qué debo hacer en su lugar?

Actualiza la plantilla y el tipo declarado para que estén sincronizados.
Verifica el tipo de la entrada y confirma si realmente se espera que sea anulable.

Si la entrada debe ser anulable, agrega `null` o `undefined` a su tipo para indicarlo.

```typescript
import { Component } from '@angular/core';

@Component({
  // Si `user` es nulo, `name` no se evaluará y la expresión devolverá
  // el valor nulo (`null` o `undefined`).
  template: `<div>{{ user?.name }}</div>`,
})
class MyComponent {
  user: { name: string } | null = { name: 'Angelino' };
}
```

Si la entrada no debe ser anulable, elimina el operador `?`.

```typescript
import { Component } from '@angular/core';

@Component({
  // La plantilla siempre muestra `name` ya que `user` garantizadamente nunca será `null`
  // ni `undefined`.
  template: `<div>{{ foo.bar }}</div>`,
})
class MyComponent {
  user: { name: string } = { name: 'Angelino' };
}
```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
[`strictNullChecks`](tools/cli/template-typecheck#strict-null-checks) también debe estar habilitado para emitir diagnósticos `optionalChainNotNullable`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "optionalChainNotNullable": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
