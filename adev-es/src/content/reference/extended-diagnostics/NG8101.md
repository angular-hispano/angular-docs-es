# Banana-in-Box Inválido

Este diagnóstico detecta una sintaxis "banana-in-box" invertida para [enlaces bidireccionales](guide/templates/two-way-binding).

```

<user-viewer ([user])="loggedInUser" />
```

## ¿Qué está mal con eso?

Tal como está, `([var])` es en realidad un [enlace de evento](guide/templates/event-listeners) con el nombre `[var]`.
El autor probablemente quiso hacer un enlace bidireccional a una variable llamada `var` pero, tal como está escrito, este enlace no tiene ningún efecto.

## ¿Qué debo hacer en su lugar?

Corrige el error tipográfico.
Como su nombre lo indica, la banana `(` debe ir _dentro_ del cuadro `[]`.
En este caso:

```

<user-viewer [(user)]="loggedInUser" />
```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`invalidBananaInBox` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "invalidBananaInBox": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
