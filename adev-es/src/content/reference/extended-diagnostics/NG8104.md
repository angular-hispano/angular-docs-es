# Atributo de texto no vinculante

Este diagnóstico garantiza que los atributos que tienen el prefijo de enlace "especial" de Angular (`attr.`, `style.` y
`class.`) sean interpretados como enlaces.

```html

<div attr.id="my-id"></div>
```

## ¿Qué está mal con eso?

En este ejemplo, `attr.id` se interpreta como un atributo regular y aparecerá
tal cual en el HTML final (`<div attr.id="my-id"></div>`). Probablemente esta no sea la intención del desarrollador.
En cambio, la intención probablemente sea establecer el atributo `id` (`<div id="my-id"></div>`).

## ¿Qué debo hacer en su lugar?

Al vincular a `attr.`, `class.` o `style.`, asegúrate de usar la sintaxis de enlace de plantilla de Angular (`[]`).

```html
<div [attr.id]="my-id"></div>
<div [style.color]="red"></div>
<div [class.large]="true"></div>
```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`textAttributeNotBinding` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "textAttributeNotBinding": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
