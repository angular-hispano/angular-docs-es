# Fusión de nulos no anulable

Este diagnóstico detecta un operador de fusión de nulos \(`??`\) innecesario en plantillas de Angular.
Específicamente, busca operaciones donde la entrada no es "anulable", lo que significa que su tipo no incluye `null` ni `undefined`.
Para dichos valores, el lado derecho de `??` nunca se utilizará.

```angular-ts
import {Component} from '@angular/core';

@Component({
  // La plantilla muestra `username` si está presente, y usa 'root' como alternativa si es
  // `null` o `undefined`.
  template: `<div>{{ username ?? 'root' }}</div>`,
})
class MyComponent {
  // `username` se declara como `string`, que _no puede_ ser `null` ni
  // `undefined`.
  username: string = 'Angelino';
}
```

## ¿Qué está mal con eso?

Usar el operador de fusión de nulos con una entrada no anulable no tiene efecto e indica una discrepancia entre el tipo permitido de un valor y cómo se presenta en la plantilla.
Un desarrollador podría asumir razonablemente que el lado derecho del operador de fusión de nulos se muestra en algún caso, pero nunca se mostrará en realidad.
Esto puede generar confusión sobre la salida esperada del programa.

## ¿Qué debo hacer en su lugar?

Actualiza la plantilla y el tipo declarado para que estén sincronizados.
Verifica el tipo de la entrada y confirma si realmente se espera que sea anulable.

Si la entrada debe ser anulable, agrega `null` o `undefined` a su tipo para indicarlo.

```ts

import {Component} from '@angular/core';

@Component({
template: `<div>{{ username ?? 'root' }}</div>`,
})
class MyComponent {
// `username` ahora es anulable. Si alguna vez se establece en `null`, se mostrará 'root'.
username: string | null = 'Angelino';
}

```

Si la entrada _no_ debe ser anulable, elimina el operador `??` y su operando derecho, ya que TypeScript garantiza que el valor siempre será no anulable.

```ts

import {Component} from '@angular/core';

@Component({
// La plantilla siempre muestra `username`, que garantizadamente nunca será `null`
// ni `undefined`.
template: `<div>{{ username }}</div>`,
})
class MyComponent {
username: string = 'Angelino';
}

```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
[`strictNullChecks`](tools/cli/template-typecheck#strict-null-checks) también debe estar habilitado para emitir diagnósticos `nullishCoalescingNotNullable`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "nullishCoalescingNotNullable": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
