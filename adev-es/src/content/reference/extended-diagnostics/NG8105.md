# Falta la palabra clave `let` en una expresión `*ngFor`

Este diagnóstico se emite cuando falta la palabra clave `let` en una expresión usada en `*ngFor`.

```typescript
import { Component } from '@angular/core';

@Component({
  // Falta la palabra clave `let` en la expresión `*ngFor`.
  template: `<div *ngFor="item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}
```

## ¿Qué está mal con eso?

La ausencia de `let` indica un error de sintaxis en la cadena de `*ngFor`. También significa que `item` no se incorporará correctamente al ámbito y `{{ item }}` no se resolverá correctamente.

## ¿Qué debo hacer en su lugar?

Agrega la palabra clave `let` que falta.

```typescript
import { Component } from '@angular/core';

@Component({
  // La palabra clave `let` está ahora presente en la expresión `*ngFor`,
  // no se emiten mensajes de diagnóstico en este caso.
  template: `<div *ngFor="let item of items">{{ item }}</div>`,
})
class MyComponent {
  items = [1, 2, 3];
}
```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`missingNgForOfLet` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "missingNgForOfLet": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
