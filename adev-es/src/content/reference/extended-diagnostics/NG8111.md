# Las funciones deben invocarse en enlaces de eventos.

Este diagnóstico detecta funciones no invocadas en enlaces de eventos.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `<button (click)="(onClick)">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## ¿Qué está mal con eso?

Las funciones en enlaces de eventos deben invocarse cuando se dispara el evento.
Si la función no se invoca, no se ejecutará cuando se dispare el evento.

## ¿Qué debo hacer en su lugar?

Asegúrate de invocar la función cuando la uses en un enlace de evento para que se ejecute cuando el evento se dispare.

```typescript
import { Component } from '@angular/core';

@Component({
  template: `<button (click)="onClick()">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## Requisitos de configuración

[`strictTemplates`](tools/cli/template-typecheck#strict-mode) debe estar habilitado para que se emita cualquier diagnóstico extendido.
`uninvokedFunctionInEventBinding` no tiene requisitos adicionales más allá de `strictTemplates`.

## ¿Qué pasa si no puedo evitar esto?

Este diagnóstico puede deshabilitarse editando el archivo `tsconfig.json` del proyecto:

```json

{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInEventBinding": "suppress"
      }
    }
  }
}
```

Consulta la [configuración de diagnósticos extendidos](extended-diagnostics#configuración) para más información.
