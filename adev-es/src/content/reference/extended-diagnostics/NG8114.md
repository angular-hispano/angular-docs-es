# Fusión de nulos sin paréntesis

Este diagnóstico detecta casos donde el operador de fusión de nulos (`??`) se combina con el operador
lógico OR (`||`) o el operador lógico AND (`&&`) sin paréntesis para desambiguar la precedencia.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `
    <button [disabled]="hasPermission() && (task()?.disabled ?? true)">
      Run
    </button>
  `,
})
class MyComponent {
  hasPermission = input(false);
  task = input<Task | undefined>(undefined);
}
```

## ¿Qué está mal con eso?

Sin paréntesis que desambigüen, es difícil entender si `??` o `||`/`&&` se
evalúa primero. Esto se considera un error en TypeScript y JavaScript, pero históricamente ha sido
permitido en las plantillas de Angular.

## ¿Qué debo hacer en su lugar?

Usa siempre paréntesis para desambiguar en estas situaciones. Si no estás seguro de cuál era la
intención original del código pero quieres mantener el mismo comportamiento, coloca los paréntesis alrededor del operador `??`.

```typescript
import { Component, signal, Signal } from '@angular/core';

@Component({
  template: `
    <button [disabled]="hasPermission() && (task()?.disabled ?? true)">
      Run
    </button>
  `,
})
class MyComponent {
  hasPermission = input(false);
  task = input<Task | undefined>(undefined);
}
```
