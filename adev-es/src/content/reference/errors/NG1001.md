# Argument Not Literal

Para hacer más rápida la extracción de metadatos en el compilador de Angular, los decoradores `@NgModule`, `@Pipe`, `@Component`, `@Directive` e `@Injectable` solo aceptan literales de objeto como argumentos.

Este es un [cambio intencional en Ivy](https://github.com/angular/angular/issues/30840#issuecomment-498869540), que impone requisitos de argumento más estrictos para los decoradores que View Engine.
Ivy requiere este enfoque porque compila los decoradores moviendo las expresiones a otras ubicaciones en la salida de la clase.

## Depurando el error

Mueve todas las declaraciones:

```ts
const moduleDefinition = {…}

@NgModule(moduleDefinition)
export class AppModule {
  constructor() {}
}

```

dentro del decorador:

```ts

@NgModule({…})
export class AppModule {
  constructor() {}
}

```
