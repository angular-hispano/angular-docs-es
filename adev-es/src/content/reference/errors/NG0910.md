# Unsafe bindings on an iframe element

Ves este error cuando Angular detecta un enlace de atributo o un enlace de propiedad en un elemento `<iframe>` usando los siguientes nombres de propiedad:

- sandbox
- allow
- allowFullscreen
- referrerPolicy
- csp
- fetchPriority

Los atributos mencionados afectan la configuración del modelo de seguridad para los `<iframe>`s
y es importante aplicarlos antes de establecer los atributos `src` o `srcdoc`.
Para garantizarlo, Angular requiere que estos atributos se establezcan en los `<iframe>`s como
atributos estáticos, de modo que los valores se establezcan en el momento de la creación del elemento y permanezcan iguales durante toda la vida útil de una instancia de `<iframe>`.

El error se lanza cuando se usa un enlace de propiedad con uno de los nombres de atributo mencionados:

```angular-html
<iframe [sandbox]="'allow-scripts'" src="..."></iframe>
```

o cuando es un enlace de atributo:

```angular-html
<iframe [attr.sandbox]="'allow-scripts'" src="..."></iframe>
```

También se lanza el error cuando se usa un patrón similar en los host bindings de una directiva:

```typescript
@Directive({
  selector: 'iframe',
  host: {
    '[sandbox]': `'allow-scripts'`,
    '[attr.sandbox]': `'allow-scripts'`,
  }
})
class IframeDirective {}
```

## Depurando el error

El mensaje de error incluye el nombre del componente con la plantilla donde se encuentra el elemento `<iframe>` con enlaces inseguros.

La solución recomendada es usar los atributos mencionados como atributos estáticos, por ejemplo:

```angular-html
<iframe sandbox="allow-scripts" src="..."></iframe>
```

Si necesitas tener valores diferentes para estos atributos (dependiendo de varias condiciones),
puedes usar un `*ngIf` o un `*ngSwitch` en un elemento `<iframe>`:

```angular-html
<iframe *ngIf="someConditionA" sandbox="allow-scripts" src="..."></iframe>
<iframe *ngIf="someConditionB" sandbox="allow-forms" src="..."></iframe>
<iframe *ngIf="someConditionC" sandbox="allow-popups" src="..."></iframe>
```
